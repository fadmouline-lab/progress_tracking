# gstack

Use the `/browse` skill from gstack for all web browsing. Never use `mcp__claude-in-chrome__*` tools.

Available gstack skills:
- `/office-hours` — async Q&A and code review
- `/plan-ceo-review` — plan review from CEO perspective
- `/plan-eng-review` — plan review from engineering perspective
- `/plan-design-review` — plan review from design perspective
- `/design-consultation` — design consultation
- `/design-shotgun` — design variants and exploration
- `/design-html` — generate production HTML/CSS from designs
- `/review` — code review
- `/ship` — ship a feature end-to-end
- `/land-and-deploy` — land and deploy changes
- `/canary` — canary deployment
- `/benchmark` — benchmarking
- `/browse` — web browsing (use this for ALL web browsing)
- `/connect-chrome` — connect to Chrome browser
- `/qa` — QA testing
- `/qa-only` — QA without code changes
- `/design-review` — design review
- `/setup-browser-cookies` — set up browser cookies
- `/setup-deploy` — set up deployment
- `/retro` — retrospective
- `/investigate` — investigate an issue
- `/document-release` — document a release
- `/codex` — Codex integration
- `/cso` — CSO review
- `/autoplan` — automatic planning
- `/plan-devex-review` — developer experience plan review
- `/devex-review` — live developer experience audit
- `/careful` — careful mode for risky operations
- `/freeze` — freeze a branch
- `/guard` — guard a branch
- `/unfreeze` — unfreeze a branch
- `/gstack-upgrade` — upgrade gstack
- `/learn` — manage project learnings
