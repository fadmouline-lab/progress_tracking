# Moulines Productivity Tracker

A lightweight project management tool built for small software teams 
who need more structure than a spreadsheet but less overhead than Jira.

My team and I (husband, wife and brother) were running a startup across WhatsApp, Notion, Figma, spreadsheets, and paper, so I built one tool to replace all of it.

Goal was to solve a real problem: scattered scope ownership, 
no visibility into who is working on what, and QA sign-off living 
in a separate doc from the work itself.

## What it does

**Scope** gives each project member their own card with reorderable 
scope bullets. Tasks are assigned per person, not dumped into one 
anonymous backlog.

**Progress** is a four-column kanban (Assigned, Working On, 
Waiting on Review, Completed) with five priority levels from 
Critical/Breaking to Can Wait (V2). Tasks link directly to their 
test checklist status.

**Test Checklist** is structured QA sign-off with core vs new-feature 
rows, batch submit, pass/fixed flows, and completed history. Lives 
next to the work, not in a separate doc.
